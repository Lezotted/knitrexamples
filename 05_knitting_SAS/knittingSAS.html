knitting SAS

As long as SAS is not in your PATH variable, you need to specify its full path with the engine.path chunk option.

Now, if you prefer to use SAS for data analysis, this shows it is possible to write the analysis in SAS, and then run:

library(knitr)
knit("knittingSAS.Rmd")

knitr is powerful enough to allow for multiple input languages in one file. Note that SAS was used in the first chunk, the following is the equivalent in R.

Thank you to Matt Mulvahill for running this example in Linux, (remove the engine.path from the chunk options if running under linux).
